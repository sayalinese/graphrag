# 知识图谱基础管理接口文档

本文档包含了 Neo4j 图谱实体的基础管理操作。这些接口是 **GraphRAG** 的底层支撑。

**Base URL Prefix:** `/api/kg`

## 1. 图谱概览与统计

### 1.1 获取图谱统计信息
- **URL**: `/api/kg/stats`
- **方法**: `GET`
- **描述**: 获取图谱的基础统计数据（节点数、关系数、存储大小等）。
- **输出范例**:
  ```json
  {
    "total_nodes": 1500,
    "total_relationships": 3200,
    "database_size": "125MB"
  }
  ```

### 1.2 获取图谱概况 (Overview)
- **URL**: `/api/kg/overview`
- **方法**: `GET`
- **描述**: 返回图谱中节点总数、关系总数以及高频节点类型分布和中心节点。
- **输出范例**:
  ```json
  {
    "nodes": 1500,
    "relationships": 3200,
    "top_labels": { "Disease": 200, "Drug": 150, "Protein": 120 },
    "central_nodes": [
      { "id": "1", "name": "肺癌", "degree": 85 }
    ]
  }
  ```

### 1.3 连接状态检查
- **URL**: `/api/kg/connection`
- **方法**: `GET`
- **描述**: 验证后端与 Neo4j 数据库的连接是否正常。
- **输出范例**:
  ```json
  {
    "status": "connected",
    "database": "neo4j",
    "version": "4.4.x"
  }
  ```

## 2. 节点与关系查询

### 2.1 获取节点列表 (分页)
- **URL**: `/api/kg/nodes`
- **方法**: `GET`
- **描述**: 分页列出图谱中的所有节点。
- **Query 参数**: `limit=50`, `type=Disease` (可选)
- **输出范例**:
  ```json
  {
    "items": [
      { "id": "1", "name": "肺癌", "labels": ["Disease"], "properties": {...} }
    ],
    "total": 1500
  }
  ```

### 2.2 搜索节点
- **URL**: `/api/kg/search`
- **方法**: `GET`
- **描述**: 根据名称或属性模糊搜索图谱中的实体。
- **Query 参数**: `q=肺癌`, `type=Disease` (可选), `limit=20`
- **输出范例**:
  ```json
  {
    "items": [
      { "id": "1", "name": "肺癌", "labels": ["Disease"], "score": 0.95 }
    ]
  }
  ```

### 2.3 获取节点详情与邻接关系
- **URL**: `/api/kg/nodes/<node_id>`
- **方法**: `GET`
- **描述**: 返回该节点的详细信息及其周边关联节点（可设定查询深度）。
- **Query 参数**: `depth=2` (默认值)
- **输出范例**:
  ```json
  {
    "node": {
      "id": "1",
      "name": "肺癌",
      "properties": { "category": "恶性肿瘤" }
    },
    "neighbors": [
      { "node": {...}, "relationship": "CAUSED_BY", "target": {...} }
    ]
  }
  ```

### 2.4 获取节点的特定关系
- **URL**: `/api/kg/nodes/<node_id>/relations`
- **方法**: `GET`
- **描述**: 查询节点的出入关系（可按方向和类型筛选）。
- **Query 参数**: `direction=OUT`, `type=CAUSES` (可选)
- **输出范例**:
  ```json
  {
    "relations": [
      { "source": "肺癌", "type": "CAUSES", "target": "咳嗽", "weight": 0.8 }
    ]
  }
  ```

### 2.5 查找最短路径
- **URL**: `/api/kg/paths/shortest`
- **方法**: `GET`
- **描述**: 计算并返回两个实体节点之间最直接的路径联系。
- **Query 参数**: `start=node_id_1`, `end=node_id_2`
- **输出范例**:
  ```json
  {
    "path": [
      { "id": "1", "name": "肺癌" },
      { "type": "CAUSES" },
      { "id": "2", "name": "咳嗽" }
    ],
    "length": 1
  }
  ```

### 2.6 查找多条路径
- **URL**: `/api/kg/paths`
- **方法**: `GET`
- **描述**: 查找两个节点之间的所有路径（限制最大长度）。
- **Query 参数**: `start=id_1`, `end=id_2`, `max_length=5`
- **输出范例**:
  ```json
  {
    "paths": [
      [ {...}, {...}, {...} ],
      [ {...}, {...} ]
    ],
    "count": 2
  }
  ```

## 3. 数据入库与管理

### 3.1 纯文本构建图谱
- **URL**: `/api/kg/upload`
- **方法**: `POST`
- **描述**: 发送一段文本，后台通过 LLM 提取三元组（实体、关系）并入库。
- **请求范例 (JSON)**:
  ```json
  {
    "text": "阿司匹林可以治疗感冒。",
    "document_title": "医学百科-条目1",
    "document_id": "doc_001"
  }
  ```
- **输出范例**:
  ```json
  {
    "success": true,
    "data": {
      "nodes_added": 3,
      "relationships_added": 2,
      "chunk_id": "chunk_001"
    }
  }
  ```

### 3.2 Excel/CSV 批量导入
- **URL**: `/api/kg/excel/import`
- **方法**: `POST`
- **描述**: 根据预设的 Schema 映射关系，将结构化表格数据批量导入图谱。
- **请求参数**: `multipart/form-data`
  - `file`: Excel/CSV 文件
  - `doc_id`: 关联的文档 ID
  - `title`: 导入标题
- **输出范例**:
  ```json
  {
    "success": true,
    "data": {
      "rows_processed": 100,
      "nodes_created": 85,
      "relationships_created": 120
    }
  }
  ```

### 3.3 获取已入库文档列表
- **URL**: `/api/kg/documents`
- **方法**: `GET`
- **描述**: 列出已被导入到图谱中的所有文档。
- **输出范例**:
  ```json
  {
    "items": [
      { "id": "doc_001", "title": "医学百科", "nodes": 50, "relationships": 80 }
    ],
    "total": 10
  }
  ```

### 3.4 获取文档分析数据
- **URL**: `/api/kg/documents/analysis`
- **方法**: `GET`
- **描述**: 获取文档在图谱中的分析数据（关键节点、节点类型分布等）。
- **Query 参数**: `limit=10` (top N 关键节点)
- **输出范例**:
  ```json
  {
    "document_id": "doc_001",
    "top_entities": [
      { "name": "肺癌", "count": 15, "type": "Disease" }
    ],
    "type_distribution": { "Disease": 20, "Drug": 15 }
  }
  ```

## 4. 社区分析与可视化

### 4.1 获取图谱社区列表
- **URL**: `/api/kg/communities`
- **方法**: `GET`
- **描述**: 获取图谱中自动划分的社区标识，可按文档筛选。
- **Query 参数**: `doc_id` (可选)
- **输出范例**:
  ```json
  {
    "communities": [
      { "id": 1, "size": 50, "key_nodes": ["肺癌", "化疗"] }
    ]
  }
  ```

### 4.2 获取图谱可视化数据
- **URL**: `/api/kg/visualize`
- **方法**: `GET`
- **描述**: 获取适合前端展示的图谱 JSON 数据（D3.js、Cytoscape 兼容）。
- **Query 参数**: `limit=100`, `doc_id` (可选), `community_id` (可选)
- **输出范例**:
  ```json
  {
    "nodes": [
      { "id": "1", "label": "肺癌", "group": 1, "size": 30 }
    ],
    "links": [
      { "source": "1", "target": "2", "label": "CAUSES", "weight": 0.8 }
    ]
  }
  ```

## 5. 图谱编辑与管理

### 5.1 更新节点属性
- **URL**: `/api/kg/nodes/<node_id>`
- **方法**: `PUT`
- **描述**: 修改节点的属性信息。
- **请求范例 (JSON)**:
  ```json
  {
    "name": "更新的名称",
    "properties": { "category": "新分类" }
  }
  ```

### 5.2 更新关系属性
- **URL**: `/api/kg/relations/<relation_id>`
- **方法**: `PUT`
- **描述**: 修改关系的权重或其他属性。
- **请求范例 (JSON)**:
  ```json
  {
    "weight": 0.9,
    "type": "RELATED_TO"
  }
  ```

### 5.3 删除节点
- **URL**: `/api/kg/nodes/<node_id>`
- **方法**: `DELETE`
- **描述**: 从图谱中删除指定节点（及其关联关系）。
- **输出范例**:
  ```json
  {
    "success": true,
    "data": { "deleted_relationships": 5 }
  }
  ```

### 5.4 删除关系
- **URL**: `/api/kg/relations/<relation_id>`
- **方法**: `DELETE`
- **描述**: 从图谱中删除指定的关系。
- **输出范例**:
  ```json
  {
    "success": true
  }
  ```
