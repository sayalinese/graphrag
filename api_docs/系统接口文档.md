# 系统运维与模式管理接口文档

**Base URL Prefix:** `/api/rag`

## 1. 模式管理 (RAG Mode)

### 1.1 获取支持的查询模式
- **URL**: `/api/rag/modes`
- **方法**: `GET`
- **描述**: 查看当前系统支持的 RAG 模式（如 `graphrag`, `vector`）。
- **输出范例**:
  ```json
  {
    "success": true,
    "data": [
      { "name": "graphrag", "description": "基于知识图谱", "enabled": true },
      { "name": "vector", "description": "基于传统向量", "enabled": false }
    ]
  }
  ```

### 1.2 切换当前全局模式
- **URL**: `/api/rag/mode`
- **方法**: `POST`
- **描述**: 在全局范围内切换默认的 RAG 检索逻辑。
- **请求范例 (JSON)**:
  ```json
  { "mode": "graphrag" }
  ```

### 1.3 获取当前模式状态
- **URL**: `/api/rag/mode`
- **方法**: `GET`

## 2. 系统调试与健康检查

### 2.1 系统健康检查
- **URL**: `/api/rag/debug/health`
- **方法**: `GET`
- **描述**: 检查后端核心组件（Neo4j 数据库、LLM API、PostgreSQL）的连接状态。
- **输出范例**:
  ```json
  {
    "success": true,
    "status": "healthy",
    "data": {
      "neo4j": "connected",
      "llm": "available",
      "timestamp": "2024-02-08T12:00:00"
    }
  }
  ```

### 2.2 获取环境配置 (仅限 Debug 模式)
- **URL**: `/api/rag/debug/config`
- **方法**: `GET`
- **描述**: 获取当前后端运行的关键配置变量（脱敏），仅在 `FLASK_DEBUG=1` 时可用。

### 2.3 获取知识库/图谱统计
- **URL**: `/api/rag/knowledge-base/stats`
- **方法**: `GET`
- **描述**: 获取图谱节点、边以及向量分块的综合统计数据。
