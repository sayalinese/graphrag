# 聊天对话与角色管理接口文档

**Base URL Prefix:** `/api/api`

## 1. 聊天对话 (Chat)

### 1.1 创建对话会话
- **URL**: `/api/api/chat/session/create`
- **方法**: `POST`
- **描述**: 初始化一个新的对话环境，可关联特定知识库。
- **请求参数 (JSON)**:
  ```json
  {
    "character_id": "student",
    "name": "答疑会话-2月8日",
    "kb_id": 1
  }
  ```
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": { "session_id": "892f2b..." },
    "message": "success"
  }
  ```

### 1.2 流式对话反馈 (SSE)
- **URL**: `/api/api/chat/stream`
- **方法**: `GET`
- **描述**: 建立 Server-Sent Events 连接，发送问题并实时接收 AI 生成的文本碎片。
- **Query 参数**:
  - `session_id`: 会话唯一标识
  - `user_message`: 用户输入的问题
- **输出流格式**:
  ```text
  data: {"type": "content", "content": "我"}
  data: {"type": "content", "content": "是"}
  data: {"type": "finish"}
  ```

### 1.3 获取历史消息
- **URL**: `/api/api/chat/history/<session_id>`
- **方法**: `GET`
- **描述**: 分页获取该会话下的所有历史问答记录。
- **Query 参数**: `page=1`, `page_size=20`
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "items": [
        { "role": "user", "content": "你好", "created_at": "..." },
        { "role": "assistant", "content": "你好！有什么可以帮助你的吗？", "created_at": "..." }
      ],
      "total": 10
    }
  }
  ```

### 1.4 删除会话
- **URL**: `/api/api/chat/session/<session_id>`
- **方法**: `DELETE`
- **描述**: 删除对话会话及其所有历史消息。
- **输出范例**:
  ```json
  {
    "code": 0,
    "message": "success"
  }
  ```

### 1.5 发送单轮消息
- **URL**: `/api/api/chat/send`
- **方法**: `POST`
- **描述**: 发送消息并等待完整回复（区别于流式，同步返回）。
- **请求范例 (JSON)**:
  ```json
  {
    "session_id": "uuid-xxx",
    "user_message": "肺癌是怎样分类的？",
    "enable_web_search": false
  }
  ```
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "response": "肺癌主要分为小细胞肺癌和非小细胞肺癌...",
      "sources": [
        { "content": "...", "source": "医学教科书" }
      ]
    }
  }
  ```

### 1.6 获取所有会话列表
- **URL**: `/api/api/chat/sessions`
- **方法**: `GET`
- **描述**: 分页列出当前用户的所有聊天会话。
- **Query 参数**: `page=1`, `page_size=10`
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "items": [
        {
          "session_id": "uuid-1",
          "name": "答疑会话-2月8日",
          "character_name": "医学生",
          "message_count": 5,
          "created_at": "..."
        }
      ],
      "total": 3
    }
  }
  ```

### 1.7 清空会话上下文缓存
- **URL**: `/api/api/chat/session/<session_id>/clear`
- **方法**: `POST`
- **描述**: 清除该会话在内存中的上下文缓存，但保留历史消息。
- **输出范例**:
  ```json
  {
    "code": 0,
    "message": "上下文已清空"
  }
  ```

### 1.8 获取单个会话详情
- **URL**: `/api/api/chat/session/<session_id>`
- **方法**: `GET`
- **描述**: 获取特定会话的配置信息（角色、知识库、消息数量等）。
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "session": {
        "session_id": "uuid-xxx",
        "name": "答疑会话-2月8日",
        "character_key": "student",
        "character_name": "医学生",
        "kb_id": 1,
        "message_count": 5,
        "created_at": "..."
      }
    }
  }
  ```

## 2. 角色管理 (Character)

### 2.1 获取所有角色列表
- **URL**: `/api/api/character/list`
- **方法**: `GET`
- **描述**: 获取系统中所有已定义的角色（包括禁用的）。
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "characters": [
        {
          "id": 1,
          "key": "student",
          "name": "医学生",
          "system_prompt": "你是一位专业的医学生...",
          "is_active": true
        }
      ]
    }
  }
  ```

### 2.2 获取可用角色
- **URL**: `/api/api/character/available`
- **方法**: `GET`
- **描述**: 仅获取启用状态的角色列表（用于前端选择）。
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "characters": [
        {
          "id": 1,
          "key": "student",
          "name": "医学生",
          "system_prompt": "你是一位专业的病理科老师..."
        }
      ]
    }
  }
  ```

### 2.3 获取角色详情
- **URL**: `/api/api/character/<key>`
- **方法**: `GET`
- **描述**: 根据角色 Key 获取该角色的详细配置。
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "character": {
        "id": 1,
        "key": "student",
        "name": "医学生",
        "system_prompt": "你是医学系学生...",
        "personality": "专业、耐心",
        "expertise": ["生理学", "病理学"],
        "is_active": true
      }
    }
  }
  ```

### 2.4 创建角色
- **URL**: `/api/api/character/create`
- **方法**: `POST`
- **描述**: 动态定义新的 AI 角色及其系统指令。
- **请求范例 (JSON)**:
  ```json
  {
    "key": "translator",
    "name": "医学翻译官",
    "system_prompt": "你是一位专业的医学英文翻译...",
    "personality": "严谨、专业",
    "expertise": ["医学术语", "文献翻译"]
  }
  ```
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "character": {
        "id": 2,
        "key": "translator",
        "name": "医学翻译官",
        "is_active": true
      }
    }
  }
  ```

### 2.5 更新角色
- **URL**: `/api/api/character/<id>`
- **方法**: `PUT`
- **描述**: 修改已有角色的配置（支持部分更新）。
- **请求范例 (JSON)**:
  ```json
  {
    "name": "高级医学生",
    "system_prompt": "更新的系统指令..."
  }
  ```

### 2.6 删除角色
- **URL**: `/api/api/character/<id>`
- **方法**: `DELETE`
- **描述**: 从系统中删除指定角色。
- **输出范例**:
  ```json
  {
    "code": 0,
    "message": "success"
  }
  ```

### 2.7 切换角色启用/禁用
- **URL**: `/api/api/character/<id>/toggle-status`
- **方法**: `POST`
- **描述**: 切换角色的启用/禁用状态，不删除数据。
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "is_active": false
    }
  }
  ```
