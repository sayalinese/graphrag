# 认证与用户管理接口文档

**Base URL Prefix:** `/api/auth` 或 `/api/user`

## 1. 认证接口 (Auth)

### 1.1 用户登录
- **URL**: `/api/auth/login`
- **方法**: `POST`
- **描述**: 用户通过用户名和密码进行身份验证。
- **请求范例 (JSON)**:
  ```json
  {
    "username": "admin",
    "password": "yourpassword"
  }
  ```
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "user": {
        "id": 1,
        "username": "admin",
        "realName": "管理员",
        "roles": ["super"]
      }
    },
    "message": "success"
  }
  ```

### 1.2 用户注册
- **URL**: `/api/auth/register`
- **方法**: `POST`
- **描述**: 注册新用户，默认角色为 `user`。
- **请求范例 (JSON)**:
  ```json
  {
    "username": "newuser",
    "password": "password123",
    "role": "user"
  }
  ```
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
       "accessToken": "...",
       "user": { ... }
    },
    "message": "success"
  }
  ```

### 1.3 获取权限码
- **URL**: `/api/auth/codes`
- **方法**: `GET`
- **描述**: 获取当前登录用户的权限角色代码。
- **Header**: `Authorization: Bearer <token>`
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": ["ROLE_USER", "ROLE_ADMIN"]
  }
  ```

## 2. 用户管理接口 (User)

### 2.1 获取个人信息
- **URL**: `/api/user/info` 或 `/api/auth/me`
- **方法**: `GET`
- **描述**: 获取当前用户的详细资料、角色及菜单权限。
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "user": {
        "id": 1,
        "username": "admin",
        "roles": [{"roleName": "Super Admin", "value": "super"}],
        "avatar": "default.png"
      }
    }
  }
  ```

### 2.2 用户列表 (管理端)
- **URL**: `/api/user/list` 或 `/api/auth/users`
- **方法**: `GET`
- **描述**: 分页查询用户列表，支持关键字搜索。
- **Query 参数**: `page=1&page_size=10&search=admin`
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "items": [...],
      "total": 5
    }
  }
  ```

### 2.3 上传头像
- **URL**: `/api/upload/avatar`
- **方法**: `POST`
- **描述**: 上传用户头像文件（png/jpg/jpeg）。
- **请求格式**: `multipart/form-data`, 字段名 `file`
- **输出范例**:
  ```json
  {
    "code": 0,
    "data": {
      "url": "/static/avatars/1_1707321234.png"
    }
  }
  ```
